# QuickRec Full v1.5 发布说明

收口日期：2026-07-10
发布状态：正式发布

## 发布定位

v1.5 是 QuickRec Full 的捕获体验与创作者工作台地基版。本版在 v1.4.1 诊断能力之上增加最近录制历史与统一入口，移除录制输出链路中的自绘光标叠加，并完成新捕获后端的技术研究，但不引入完整工作台或替换默认 dxcam 链路。

## 主要变化

### 最近录制

- 在视频保存目录的 `QuickRecMetadata\recordings.json` 中维护本地录制历史。
- 默认保留最近 50 条记录，新记录显示在顶部。
- 托盘空闲/录制中菜单和录制完成结果条均可打开最近录制窗口。
- 支持打开文件、打开所在目录、复制完整路径和从列表移除索引。
- 文件被移动或删除时显示缺失状态；无历史时显示明确空状态。
- 历史读取或写入失败不会把已成功保存的视频误报为录制失败。

### 捕获体验

- 全屏、区域和窗口录制输出帧不再叠加 QuickRec 自绘光标。
- 鼠标点击高亮继续作为录制时的桌面实时提示，不额外写入输出视频。
- 默认捕获后端仍为 dxcam；WGC 仅形成研究结论，不进入本版运行时。

### 原型与文档

- Full 原型补充最近录制、素材入库、缺失记录、空状态和模拟操作。
- PRD、开发计划、进度、手动验收、Bugfix、捕获后端研究和发布文档统一收纳在本目录。

## 已知限制

- 最近录制仅维护当前保存路径下的本地 JSON 索引，不扫描其他目录，不使用数据库，也不提供云同步。
- 历史默认最多 50 条；从列表移除只删除索引，不删除视频文件。
- 当前不在视频中绘制 QuickRec 自绘光标，也未引入可原生捕获光标的新后端。
- WGC 仍处于 spike 研究阶段；默认 dxcam、窗口移动冻结策略和既有捕获边界保持不变。
- Full 高保真原型是产品设计与验收资料，不是应用运行或发布依赖。
- 真实桌面、托盘和音频设备仍不能完全由 CI 覆盖，后续发布仍需硬件与 GUI 验收。

## 回滚方式

- 代码回滚点：`v1.4.1` tag，固定指向提交 `16c7dce`。
- 发布包回滚：停止 v1.5 进程，恢复 v1.4.1 稳定包，并保留 v1.5 生成的视频文件。
- `QuickRecMetadata\recordings.json` 是附加索引；回滚到 v1.4.1 后可保留该文件，不影响旧版录制主链路。
- 不移动或重写 `v1.4.1` tag。
