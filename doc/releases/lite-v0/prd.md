> 版本：QuickRec Lite v0  
> 基线：QuickRec v1.4 稳定发布基线  
> 分支：lite  
> 日期：2026-07-08  
> 状态：规划中 / 待开发

## 1. 产品概述和目标用户

### 1.1 产品概述

QuickRec Lite 是 QuickRec 从 v1.4 稳定基线拆分出的**独立轻量级录屏工具**。Lite 不是 QuickRec Full 的免费版、低配模式或运行时功能开关，而是一条长期保持简单、低心智负担和较小依赖的独立产品线。

Lite v0 面向普通用户，提供打开即用的一键全屏录制体验。用户无需理解区域录制、窗口录制、捕获后端、编码参数或工作台概念，只需要选择是否录制声音，然后开始录制。核心定位：比系统自带录屏更直接，比完整 QuickRec 更轻、更少选项、更少干扰。

### 1.2 目标用户

| 用户类型 | 使用场景 | 频率 | Lite v0 关注点 |
|---------|---------|------|----------------|
| 普通用户 | 录制屏幕内容、网页内容、简单操作过程 | 偶尔 | 不学习复杂设置，直接开始录制 |
| 学生/轻办公用户 | 录制简单说明、课堂片段、软件操作 | 每周 1-3 次 | 保留系统声和麦克风，输出可直接分享 |
| 非专业创作者 | 录制短视频素材或说明片段 | 偶尔 | 原生画质、60fps、文件可播放 |

Lite v0 不优先覆盖专业教程制作者、游戏录制用户、多窗口工作流用户和需要素材管理/导出队列的创作者。这些能力归入 QuickRec Full 方向。

### 1.3 产品目标

- **一键开始全屏录制**：主窗口、托盘或快捷键均可快速开始录制。
- **高质量默认值**：固定原生分辨率和 60fps，不要求用户理解画质档位。
- **音频完整但选择简单**：保留无声、系统声音、麦克风、两者都有四种音频模式。
- **功能少而稳定**：不提供区域录制、窗口录制、鼠标点击高亮和倒计时。
- **体积持续优化**：目标打包产物低于 200MB，但体积不作为 Lite v0 发布阻断项。
- **可独立演进**：Lite 与 Full 后续作为不同产品方向演进，不把 Full 的复杂能力回灌到 Lite。

## 4. 界面设计要求

### 4.1 设计原则

- **沿用现有轻量窗口**：Lite v0 不重新设计主窗口、托盘、工具栏和设置窗口。
- **只移除不可用入口**：被裁剪功能直接从 UI 移除，不置灰、不提示升级、不做 Full 引流。
- **少设置项**：设置窗口只保留 Lite v0 可用能力。
- **点击即录**：移除倒计时后，开始录制应尽快进入实际录制状态。

### 4.2 主窗口

主窗口保留当前完整版入口形态，但只展示全屏录制相关操作。

```
┌──────────────────────────────┐
│ QuickRec Lite                │
│                              │
│        ▶ 开始录制             │
│                              │
│ 音频：无 / 系统声 / 麦克风 / 两者 │
│ 保存路径：C:\Users\...\Videos │
│                              │
│ ⚙ 设置        📁 打开文件夹     │
└──────────────────────────────┘
```

要求：

- 不展示区域录制入口。
- 不展示窗口录制入口。
- 不展示点击高亮入口。
- 不展示倒计时入口。
- 文案使用 `QuickRec Lite` 或等价轻量版本标识。

### 4.3 系统托盘菜单

**空闲状态：**

```
┌──────────────────┐
│  ▶ 全屏录制       │
│  ⚙ 设置          │
│  📁 打开保存文件夹 │
│  ─────────────    │
│  ✕ 退出           │
└──────────────────┘
```

**录制中：**

```
┌──────────────────┐
│  ⏸ 暂停/继续      │
│  ⏹ 停止录制       │
│  ─────────────    │
│  ✕ 退出           │
└──────────────────┘
```

要求：

- 移除区域录制菜单项。
- 移除窗口录制菜单项。
- 录制中菜单延续现有动态菜单逻辑。

### 4.4 设置窗口

Lite v0 设置项：

```
┌──────────────────────────────────────────────┐
│ QuickRec Lite 设置                            │
├──────────────────────────────────────────────┤
│ 保存路径    [C:\Users\xxx\Videos\QuickRec]    │
│ 音频源      ● 无  ○ 系统声音                   │
│             ○ 麦克风  ○ 两者都有               │
│ 快捷键      开始: [Ctrl+Shift+R] 点击修改...   │
│             停止: [Ctrl+Shift+S] 点击修改...   │
│             暂停: [Ctrl+Shift+P] 点击修改...   │
│ □ 开机自启                                   │
│                                              │
│              [保存]  [取消]                   │
└──────────────────────────────────────────────┘
```

设置窗口不再展示：

- 画质选择。
- 帧率选择。
- 区域录制快捷键。
- 窗口录制快捷键。
- 录制倒计时。
- 鼠标高亮开关。

### 4.5 录制工具栏和结果条

录制工具栏和结果条保留当前完整版交互：

- 录制时显示计时、暂停、停止、取消。
- 暂停时显示恢复状态。
- 保存中禁用操作按钮。
- 保存完成后显示结果条。
- 结果条可打开文件和打开文件夹。

倒计时 UI 不进入 Lite v0。

## 5. 技术架构和依赖边界

### 5.1 技术选型

| 模块 | 技术 | Lite v0 要求 |
|------|------|--------------|
| 屏幕捕获 | dxcam + comtypes | 保留现有稳定链路 |
| 视频编码 | FFmpeg pipe + libx264 | 保留 H.264 MP4 输出 |
| 图像处理 | OpenCV / Pillow | 允许评估 headless 或替代方案，但不得破坏捕获链路 |
| UI | PyQt5 | 保留现有窗口体系 |
| 系统托盘 | pystray | 保留 |
| 快捷键 | pynput | 保留开始/暂停/停止 |
| 系统声音 | soundcard | 保留 |
| 麦克风 | pyaudio | 保留 |
| 通知 | winotify | 保留 |
| 打包 | PyInstaller | 保留单目录可运行产物 |

### 5.2 配置边界

Lite v0 配置应收敛为：

| 配置项 | Lite v0 |
|--------|---------|
| `save_path` | 保留 |
| `audio_source` | 保留 |
| `shortcut_start` | 保留 |
| `shortcut_stop` | 保留 |
| `shortcut_pause` | 保留 |
| `auto_start` | 保留，默认关闭 |
| `quality` | 固定原生，不在 UI 暴露 |
| `fps` | 固定 60，不在 UI 暴露 |
| `shortcut_area` | 不在 UI 暴露 |
| `shortcut_window` | 不在 UI 暴露 |
| `show_countdown` | 移除或固定关闭 |
| `mouse_highlight` | 移除或固定关闭 |

### 5.3 裁剪策略

Lite v0 采用两阶段策略：

1. **v0 阶段**：先移除用户入口和设置项，底层区域/窗口录制代码暂不删除。
2. **v0.1 阶段**：在 v0 验证稳定后，再评估删除、隔离或迁出区域/窗口录制代码。

该策略的目标是避免一次性删除大量代码导致全屏录制和音频链路回归，同时让用户可见能力先符合 Lite 定位。

## 6. 性能、体积和质量指标

### 6.1 性能指标

| 指标 | Lite v0 要求 |
|------|--------------|
| 启动到可录制 | 3 秒内进入可操作状态 |
| 录制启动延迟 | 点击开始后尽快进入录制，不显示倒计时 |
| 帧率 | 固定 60fps，实际输出应接近目标帧率 |
| 分辨率 | 主显示器原生分辨率 |
| 输出格式 | MP4 / H.264，可被系统播放器打开 |

### 6.2 体积指标

| 指标 | Lite v0 要求 |
|------|--------------|
| 目标体积 | 低于 200MB |
| 阻断规则 | 未低于 200MB 不阻断 v0 发布 |
| 必需记录 | 产物体积、主要依赖构成、优化尝试、未达标原因 |

### 6.3 质量门槛

发布前必须通过：

```powershell
python -m compileall src scripts tests
python -m ruff check .
python -m mypy
python -m pytest -q
```

如果裁剪入口导致历史测试期望变化，应同步更新测试用例，确保测试表达 Lite v0 的真实产品边界。

## 8. 版本边界和后续路线

### 8.1 Lite 与 Full 的边界

| 方向 | Lite | Full |
|------|------|------|
| 产品定位 | 一键轻量全屏录制 | 创作者工作台 |
| 录制模式 | 全屏 | 全屏、区域、窗口及后续扩展 |
| 音频 | 四种基础音频模式 | 完整音频链路和后续高级能力 |
| UI | 沿用轻量窗口，减少入口 | 多页面工作台 |
| 高级能力 | 不承接 | 历史、素材、轻编辑、导出、诊断 |
| 高帧率 | 不进入 v0 | 可作为实验能力 |

### 8.2 后续路线

| 版本 | 方向 |
|------|------|
| Lite v0 | 建立只保留全屏录制的轻量基线，完成用户可见功能裁剪和发布验收 |
| Lite v0.1 | 根据 v0 验证结果继续推进体积优化和依赖边界收敛 |
| Lite v1.0 | 在稳定反馈基础上决定是否彻底删除区域/窗口底层代码，或恢复少量可选能力 |

## 9. 风险和缓解策略

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| 原生 60fps 对低性能机器压力较高 | 可能掉帧、CPU 占用升高 | v0 记录风险，不提供降档；后续根据反馈评估 |
| 双音频混流受设备环境影响 | 可能初始化失败或混流失败 | 保留音频自检与降级，发布前四种音频模式必测 |
| 体积低于 200MB 不确定 | FFmpeg、OpenCV、PyQt5、Python runtime 体积较大 | 体积作为优化目标，不作为阻断验收 |
