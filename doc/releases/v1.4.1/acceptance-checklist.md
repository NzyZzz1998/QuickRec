## 6. C 复制诊断信息

### 操作步骤

1. 打开设置页。
2. 点击：

```text
复制诊断信息
```

3. 打开记事本或任意文本框，粘贴剪贴板内容。

### 预期内容

诊断文本应包含类似信息：

```text
QuickRec Diagnostic Report
app
config
recorder
ffmpeg
audio
window
recent_logs
```

| 编号  | 结论要求                          | 结论  | 证据    |
| --- | ----------------------------- | --- | ----- |
| C1  | 点击后应用不崩溃                      | 通过  | 说明或截图 |

| C3  | 文本包含 FFmpeg 路径、保存路径、音频配置、最近日志 | 通过  | 粘贴内容  |
| C4  | 中文不乱码                         | 通过  | 粘贴内容  |

## 12. I 异常场景验收

至少完成 1 个异常场景。推荐优先做 FFmpeg 缺失场景。

### 推荐场景：FFmpeg 缺失

操作步骤：

1. 退出 QuickRec。
2. 找到：

```text
E:\codex\QuickRec\dist\QuickRec\_internal\ffmpeg\ffmpeg.exe
```

3. 临时改名为：

```text
ffmpeg.exe.bak
```

4. 启动 QuickRec。
5. 尝试录制一次。
6. 打开设置页，点击“复制诊断信息”或“导出诊断文件”。
7. 验证诊断信息里能看到 FFmpeg 缺失相关上下文。
8. 验收后把文件名改回：

```text
ffmpeg.exe
```

### 预期诊断内容

诊断文本应包含类似字段：

```text
ffmpeg
ffmpeg_path
exists: False
failure
error
```
