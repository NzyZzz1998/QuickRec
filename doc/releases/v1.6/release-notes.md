## 版本定位

v1.6 将 v1.5 的“最近录制”升级为跨保存路径的轻量素材库基础版。它建立统一素材身份、中央索引、迁移恢复和安全文件操作能力，为后续完整创作者工作台提供可靠数据基础，但本版不包含项目管理、剪辑、导出队列或 AI 功能。

## 主要变化

- 新增 `%APPDATA%\QuickRec\recordings.json` 中央素材索引，最多保留最近 200 条记录。
- 托盘和录制结果条中的“最近录制”统一升级为“素材库”。
- 素材库默认展示 50 条，每次加载更多 50 条，并提供列表与详情视图。
- 支持 v1.5 旧索引首次迁移、其他目录手动导入和当前层级目录重建。
- 支持索引有效备份、损坏归档、自动恢复和恢复失败后的目录重建。
- 支持缺失素材重新定位，稳定素材 ID 和原始录制时间保持不变。
- 新录制直接写入时长、输出分辨率、FPS、模式、音频来源和文件大小。
- 打包加入同版本 `ffprobe.exe`，用于旧素材元数据补齐和结构化失败降级。
- 支持从素材库仅移除索引，或将单个 MP4 移入 Windows 回收站。
- 索引写入失败不会把已保存的视频误报为录制失败，并提供重试入库入口。

## 已知限制

- 使用本地 JSON 索引，不使用数据库、云同步或跨设备同步。
- 素材库最多保留 200 条索引；淘汰索引不会删除视频。
- 目录重建不递归，只扫描当前层级 `QuickRec_*.mp4`。
- 不提供搜索、筛选、标签、收藏、内嵌预览或批量删除。
- 索引写入失败后的结果条提供短时重试；结果条关闭后的持久恢复入口延后至后续版本，不阻塞本版发布。
- WGC、120 FPS、多显示器正式支持、完整工作台和 AI 均不在本版范围。

## 回滚方式

- 代码回滚点：`v1.5` tag。
- 发布包回滚：继续使用 QuickRec Full v1.5 Release 资产。
- v1.5 旧索引在迁移后原样保留；回滚不会依赖或重写 v1.6 中央索引。
