# QuickRec Full v1.6 变更日志

> 状态：正式发布
> 日期：2026-07-13

## Added

- 中央素材索引与稳定素材 ID。
- 素材库列表、详情、加载更多和多状态反馈。
- v1.5 历史迁移、手动导入和目录重建。
- 索引备份、损坏归档和自动恢复。
- 单项重新定位与批量候选确认。
- 会话已知元数据和内置 `ffprobe` 元数据补齐。
- Windows 回收站删除边界。
- 索引失败独立提示与结果条重试入库。

## Changed

- 托盘和结果条的“最近录制”入口统一更名为“素材库”。
- 新录制从保存路径分散索引切换为 `%APPDATA%\QuickRec\recordings.json` 中央索引。
- 诊断版本号改由 `src/version.py` 单一事实源提供。
- 开机自启测试改为隔离注册表 mock，不再读写真实 HKCU 启动项。

## Fixed

- 修复诊断报告长期显示旧版 `v1.4.x` 的版本事实错误。
- 修复 Run 注册表键不存在时无法安全创建开机自启项的问题。
- 修复素材刷新后抢占当前列表选择的问题。
- 修复打包环境中 FFprobe 输出解码导致有效 MP4 元数据缺失的问题。
- 修复重新定位接受损坏 MP4 且失败后无法继续纠正的问题。
- 修复无备份损坏索引确认重建后无法提交的问题。
- 修复素材库空状态仍显示上一条详情的问题。

## Compatibility

- v1.5 保存目录中的 `QuickRecMetadata\recordings.json` 保持原样，不删除、不重命名。
- `v1.5` tag 和发布包继续作为稳定回滚点。
- QuickRec Lite 未修改。
