# QuickRec - 轻量级录屏工具 PRD

## 1. 产品概述和目标用户

### 1.1 产品概述

QuickRec 是一款**轻量级、零配置**的 Windows 桌面录屏工具。用户打开即用，一键开始录制，自动保存到指定文件夹，无需登录、无需云同步、无多余功能。核心定位：比系统自带录屏更灵活，比专业录屏软件更简单。

### 1.2 目标用户

| 用户类型 | 使用场景 | 频率 |
|---------|---------|------|
| 办公人群 | 录制操作教程、bug 复现步骤、会议要点 | 每周 2-5 次 |
| 教师/培训师 | 录制课件讲解、操作演示 | 每周 1-3 次 |
| 开发者 | 录制 bug 复现、功能演示 | 偶尔 |
| 普通用户 | 录制游戏片段、生活记录 | 偶尔 |

### 1.3 产品目标

- **3 秒内开始录制**：从点击录制到实际录制不超过 3 秒
- **零学习成本**：首次使用无需教程即可完成录制
- **安装包 < 50MB**：轻量不占资源

## 4. 界面设计要求

### 4.1 设计原则

- **极简**：界面元素不超过 5 个按钮
- **半透明悬浮**：录制工具栏始终置顶，半透明不遮挡太多内容
- **系统托盘**：非录制状态下隐藏到系统托盘

### 4.2 主界面（录制工具栏）

```
┌──────────────────────────────────────────────┐
│  ● 00:03:25  │  ⏸ 暂停  │  ⏹ 停止  │  ✕ 取消 │
└──────────────────────────────────────────────┘
```

- 状态：红色圆点（`#e74c3c`）+ 计时器（如 `00:03:25`，Consolas 等宽字体）
- 暂停时：黄色圆点（`#f39c12`），按钮文字变为"继续"
- 保存中：蓝色圆点（`#3498db`），文字显示"保存中..."，按钮全部禁用
- 仅 4 个按钮：暂停 / 停止 / 取消
- 可拖动位置，默认在屏幕顶部中央偏上（距顶部 10px）
- 背景色：`rgba(26, 26, 46, 230)`，圆角 8px

### 4.3 系统托盘菜单

```

│  ▶ 开始录制       │

│  📁 打开保存文件夹 │
```

## 5. 技术架构

### 5.1 技术栈

| 层级 | 技术 | 版本 | 用途 |
|-----|------|------|------|
| 语言 | Python | 3.12.8 | 主开发语言 |
| UI 框架 | PyQt5 | 5.15+ | 桌面界面开发 |
| 屏幕捕获 | dxcam | 0.3+ | DirectX 高性能屏幕截图（约5ms/帧） |
| 视频编码 | OpenCV | 4.8+ | 将帧序列编码为 MP4 |
| 数值计算 | NumPy | 1.24+ | 图像数据处理 |
| 热键 | pynput | 1.7+ | 全局快捷键监听（无需管理员权限） |
| 系统托盘 | pystray | 0.19+ | 系统托盘图标 |
| 打包 | PyInstaller | 6.0+ | 打包为 .exe |

### 5.2 架构设计

```
main.py（程序入口，线程安全信号桥协调）
├── config.py（配置管理）
├── ui/tray_icon.py（系统托盘，_SignalBridge 信号桥）
├── ui/toolbar.py（录制工具栏）
├── ui/settings_dialog.py（设置对话框，_ShortcutRecorder）
├── hotkey/hotkey_manager.py（全局快捷键，pynput 字符串标识符匹配）
├── recorder/recorder_manager.py（录制控制器，临时文件缓存 + 后编码）
│   ├── recorder/screen_capturer.py（屏幕捕获，dxcam 延迟初始化）
│   ├── recorder/video_encoder.py（视频编码，OpenCV mp4v）
│   ├── utils/file_namer.py（文件自动命名）
│   └── utils/disk_checker.py（磁盘空间检查）
└── 三处信号桥：Tray/Hotkey/Saved → Qt 主线程
```

### 5.3 核心设计决策

| 决策 | 原因 |
|-----|------|
| 临时文件缓存方案 | 录制时 JPEG 压缩写临时文件（~7ms/帧），停止后后台解码写 MP4；内存始终 MB 级 |
| dxcam 延迟初始化 | dxcam.create() 和 camera.start() 在录制线程中执行，避免主线程冻结 |
| time.sleep 替代忙等待 | 录制循环帧间等待用 time.sleep 释放 GIL，避免主线程冻结 |
| 非阻塞 stop() | stop() 立即返回，join + 清理 + 编码启动在后台线程 _stop_and_encode() 中完成 |
| 三处信号桥 | pyqtSignal 机制将 pystray/pynput/编码线程的回调安全转发到 Qt 主线程 |

### 5.4 打包

```bash
# 必须使用标准 CPython 3.12.8，不能用 Anaconda（DLL 布局不兼容）
cd E:\CC_Learning\QuickRec_dev
D:\Work\Software\Python\Scripts\pyinstaller.exe build_std.spec --noconfirm
# 输出: dist/QuickRec/QuickRec.exe
```

## 6. 非功能性需求

### 6.1 性能需求

| 指标 | 要求 | v1.0 实际值 |
|-----|------|-----------|
| 启动时间 | < 2 秒 | ~1 秒 |
| CPU 占用（录制状态） | < 15% | ~5%（30fps）/ ~10%（60fps） |
| 内存占用（录制状态） | < 200MB | ~80MB |
| 内存占用（空闲状态） | < 100MB | ~45MB |
| 录制延迟（点击到开始） | < 500ms | ~200ms |
| 文件大小（原生2K/30fps） | < 100MB/分钟 | ~35MB/分钟 |
| 文件大小（1080p/30fps） | < 80MB/分钟 | ~25MB/分钟 |

### 6.2 兼容性需求

| 平台 | 最低版本 | 说明 |
|-----|---------|------|
| Windows | Windows 10 (1903+) | 仅支持 x64，仅录制主显示器 |

> v1.0 仅支持 Windows，macOS/Linux 支持计划在 v2.0+ 考虑。

### 6.3 安全与隐私

| 要求 | 说明 |
|-----|------|
| 本地存储 | 所有录制内容仅保存在用户本地，不上传任何数据 |
| 无账号系统 | 不收集用户信息，不要求注册/登录 |
| 权限最小化 | 仅在录制时捕获屏幕，录制结束后释放资源 |

### 6.4 已知限制

| 限制 | 说明 | 计划 |
|-----|------|------|
| 仅全屏录制 | 区域录制因 Windows 11 兼容性问题推迟 | v1.1 |
| 无音频录制 | 不支持系统声音和麦克风录制 | v1.1 |
| 单显示器 | 仅录制主显示器 | v2.0 |
| 编码格式 | 使用 OpenCV mp4v，画质不如 x264 | v1.1 可切换 x264 |

## 8. 风险与应对

| 风险 | 影响 | 应对 |
|-----|------|------|
| Windows 11 区域选择器点击穿透 | 区域录制功能不可用 | 推迟到 v1.1，移除 Qt.Tool 标志 |
| pynput 与 Qt 线程冲突 | 快捷键回调冻结 GUI | pyqtSignal 信号桥转发到主线程 |
| 录制线程占用 GIL | 主线程冻结 | time.sleep 替代忙等待，释放 GIL |
| dxcam 初始化阻塞主线程 | 开始录制时短暂卡顿 | 延迟到录制线程初始化 |
| mp4v 编码质量不如 x264 | 视频画质一般 | v1.1 可切换 x264 编码 |
| 打包后体积较大 | 安装包超 50MB | 可考虑 UPX 压缩或排除不必要依赖 |
