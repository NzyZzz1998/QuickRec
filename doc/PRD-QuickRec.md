# QuickRec - 轻量级录屏工具 PRD

## 1. 产品概述和目标用户

### 1.1 产品概述

QuickRec 是一款**轻量级、零配置**的桌面录屏工具。用户打开即用，一键开始录制，自动保存到指定文件夹，无需登录、无需云同步、无多余功能。核心定位：比系统自带录屏更灵活，比专业录屏软件更简单。

### 1.2 目标用户

| 用户类型 | 使用场景 | 频率 |
|---------|---------|------|
| 办公人群 | 录制操作教程、bug 复现步骤、会议要点 | 每周 2-5 次 |
| 教师/培训师 | 录制课件讲解、操作演示 | 每周 1-3 次 |
| 开发者 | 录制 bug 复现、功能演示 | 偶尔 |
| 普通用户 | 录制游戏片段、生活记录 | 偶尔 |

### 1.3 产品目标

- **3 秒内开始录制**：从点击录制到实际录制不超过 3 秒
- **零学习成本**：首次使用无需教程即可完成录制
- **安装包 < 50MB**：轻量不占资源

## 2. 详细功能列表

### 2.1 核心录制功能

| 编号 | 功能 | 说明 |
|-----|------|------|
| F1 | 全屏录制 | 录制整个屏幕 |
| F2 | 区域录制 | 用户拖拽选择矩形区域进行录制 |
| F3 | 指定窗口录制 | 点击某个窗口，只录制该窗口内容 |
| F4 | 系统声音录制 | 同时录制电脑发出的声音 |
| F5 | 麦克风录制 | 同时录制麦克风声音（解说用） |
| F6 | 无声音录制 | 仅录制画面，不录声音 |
| F7 | 开始/暂停/停止 | 录制过程中可暂停和恢复 |

### 2.2 保存功能

| 编号 | 功能 | 说明 |
|-----|------|------|
| S1 | 自定义保存路径 | 用户可设置默认保存文件夹 |
| S2 | 自动命名 | 按 `QuickRec_YYYYMMDD_HHmmss.mp4` 格式自动命名 |
| S3 | MP4 格式输出 | 通用格式，无需转码即可播放和分享 |
| S4 | 录制完成提示 | 录制完成后弹出通知，可快速打开文件所在文件夹 |

### 2.3 快捷键

| 编号 | 功能 | 默认快捷键 |
|-----|------|-----------|
| K1 | 开始录制 | `Ctrl + Shift + R` |
| K2 | 暂停/继续 | `Ctrl + Shift + P` |
| K3 | 停止录制 | `Ctrl + Shift + S` |
| K4 | 自定义快捷键 | 设置中可修改 |

### 2.4 设置功能

| 编号 | 功能 | 说明 |
|-----|------|------|
| P1 | 保存路径设置 | 选择默认保存文件夹 |
| P2 | 画质选择 | 高（1080p）/ 中（720p）/ 低（480p） |
| P3 | 帧率选择 | 30fps / 60fps |
| P4 | 音频源选择 | 系统声音 / 麦克风 / 无 / 两者都有 |
| P5 | 快捷键设置 | 修改录制快捷键 |
| P6 | 开机自启 | 可选是否开机自动启动（静默状态） |
| P7 | 录制倒计时 | 可选 3 秒倒计时或立即开始 |

## 4. 界面设计要求

### 4.1 设计原则

- **极简**：界面元素不超过 5 个按钮
- **半透明悬浮**：录制工具栏始终置顶，半透明不遮挡太多内容
- **系统托盘**：非录制状态下隐藏到系统托盘

### 4.2 主界面（录制工具栏）

```
┌──────────────────────────────────────────────┐
│  ● REC  │  ⏸ 暂停  │  ⏹ 停止  │  ✕ 取消    │
└──────────────────────────────────────────────┘
```

- 状态：红色圆点 + 计时器（如 `● 00:03:25`）
- 仅 4 个按钮：暂停 / 停止 / 取消
- 可拖动位置，默认在屏幕顶部中央

### 4.3 选择区域界面

```
+----------------------------------+
|                                  |
|    ┌──────────────────┐          |
|    │                  │          |
|    │   拖拽选择区域    │          |
|    │                  │          |
|    └──────────────────┘          |
|                                  |
|    [ 开始录制 ]  [ 取消 ]         |
+----------------------------------+
```

- 半透明遮罩覆盖全屏
- 中间透明区域为选中录制区域
- 拖拽时实时显示区域尺寸（如 `1280 x 720`）

### 4.4 设置界面

```
┌─ QuickRec 设置 ────────────────────┐
│                                    │
│  保存路径    [C:\Videos\QuickRec]  │
│                                    │
│  画质        ● 高(1080p) ○ 中 ○ 低 │
│                                    │
│  音频源      ○ 系统声音            │
│              ○ 麦克风              │
│              ○ 两者都有            │
│              ○ 无                  │
│                                    │
│  快捷键      [Ctrl+Shift+R] 修改... │
│                                    │
│              [ 保存 ]  [ 取消 ]     │
└────────────────────────────────────┘
```

### 4.5 系统托盘菜单

```
┌──────────────────┐
│  ▶ 开始录制       │
│  ⚙ 设置          │
│  📁 打开保存文件夹 │
│  ─────────────    │
│  ✕ 退出           │
└──────────────────┘
```

### 4.6 视觉风格

- 深色主题工具栏（`#1a1a2e` 半透明，圆角 8px）
- 录制状态按钮用红色（`#e74c3c`），其他用灰色
- 计时器用等宽字体，白色
- 整体风格参考 Windows 11 的 Fluent Design

## 5. 技术栈建议

### 5.1 推荐方案：Electron + TypeScript

| 层级 | 技术 | 理由 |
|-----|------|------|
| 框架 | Electron 32+ | 跨平台，原生 API 调用成熟 |
| 语言 | TypeScript | 类型安全，维护性好 |
| UI | React + Tailwind CSS | 开发效率高，组件化 |
| 录制引擎 | `electron.screenCapture` + `ffmpeg` | Electron 原生 API + FFmpeg 编码 |
| 音频 | `desktopCapturer` + `MediaRecorder` | 系统/麦克风声音分离 |
| 打包 | electron-builder | 一键打包为 exe/dmg |

### 5.2 备选方案 A：Tauri + Rust

| 优势 | 劣势 |
|-----|------|
| 安装包 < 10MB | Rust 学习曲线陡 |
| 内存占用更低 | 跨平台音频处理较复杂 |
| 性能更好 | 生态不如 Electron 成熟 |

### 5.3 备选方案 B：Python + PyQt

| 优势 | 劣势 |
|-----|------|
| 开发速度最快 | 打包后体积大（~100MB） |
| 生态丰富 | Windows 上音频处理有坑 |
| 适合快速 MVP 验证 | 跨平台兼容性一般 |

### 5.4 核心依赖（Electron 方案）

```json
{
  "dependencies": {
    "electron": "^32.0.0",
    "react": "^18.0.0",
    "ffmpeg-static": "^5.2.0",
    "fluent-ffmpeg": "^2.1.2"
  },
  "devDependencies": {
    "typescript": "^5.0.0",
    "tailwindcss": "^3.4.0",
    "electron-builder": "^24.0.0"
  }
}
```

### 5.5 关键 API

- `navigator.mediaDevices.getDisplayMedia()` - 屏幕捕获
- `desktopCapturer` - 获取屏幕/窗口列表
- `MediaRecorder` - 视频流编码
- `child_process` - 调用 FFmpeg 进行后处理

## 6. 非功能性需求

### 6.1 性能需求

| 指标 | 要求 | 说明 |
|-----|------|------|
| 启动时间 | < 2 秒 | 冷启动到可录制状态 |
| CPU 占用 | < 15% | 1080p/30fps 录制时 |
| 内存占用 | < 200MB | 录制状态下 |
| 录制延迟 | < 500ms | 点击录制到实际开始录制 |
| 文件大小 | < 100MB/分钟 | 1080p/30fps 参考值 |

### 6.2 兼容性需求

| 平台 | 最低版本 | 要求 |
|-----|---------|------|
| Windows | Windows 10 (1903+) | 支持 x64 |
| macOS | macOS 12+ | 支持 Apple Silicon 和 Intel |
| Linux | Ubuntu 20.04+ | 支持 Wayland 和 X11 |

### 6.3 安全与隐私

| 要求 | 说明 |
|-----|------|
| 本地存储 | 所有录制内容仅保存在用户本地，不上传任何数据 |
| 无账号系统 | 不收集用户信息，不要求注册/登录 |
| 权限最小化 | 仅在录制时申请屏幕录制权限，录制结束后释放 |
| 透明告知 | 录制时显示明显指示器，防止偷录 |

### 6.4 可用性

| 要求 | 说明 |
|-----|------|
| 无障碍 | 支持键盘操作，不依赖鼠标 |
| 高 DPI | 支持 4K 屏幕，界面不模糊 |
| 多语言 | 首发支持中文和英文 |
| 异常处理 | 磁盘空间不足时提前警告 |

### 6.5 可靠性

| 要求 | 说明 |
|-----|------|
| 崩溃恢复 | 录制过程中崩溃，已录制部分尽量保留 |
| 磁盘保护 | 磁盘满时自动停止录制，不导致系统崩溃 |
| 权限异常 | 无录制权限时友好提示用户开启权限 |

## 8. 风险与应对

| 风险 | 影响 | 应对 |
|-----|------|------|
| Windows/macOS 录屏权限机制不同 | 跨平台兼容困难 | 优先开发 Windows 版本，macOS 后续跟进 |
| FFmpeg 打包体积大 | 安装包超标 | 使用 ffmpeg-static，按需裁剪 |
| 高分屏录制性能差 | CPU/内存超标 | 限制最高 1080p，提供低画质选项 |
| 系统更新导致 API 变化 | 功能失效 | 定期测试，保持 Electron 版本更新 |
