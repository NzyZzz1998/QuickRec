# QuickRec v1.4 测试用例

> 版本: v1.4
> 创建时间: 2026-07-04
> 测试环境: Windows 10/11, Python 3.12.8, PyQt5
> 前置条件: v1.3 已有功能保持可用
> 测试方式: 自动化测试 + PyCharm 运行 `src/main.py` + PyInstaller 打包后冒烟验证

---

## 1. 测试环境准备

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows 10/11 |
| Python | CPython 3.12.x |
| 依赖 | `python -m pip install -r requirements-dev.txt` |
| FFmpeg | 打包前音视频验证和打包验证需要 `ffmpeg/ffmpeg.exe` 就位 |
| 开发运行 | 在 PyCharm 中以项目根目录为 working directory 运行 `src/main.py` |
| 打包命令 | `python -m PyInstaller build_std.spec --clean --noconfirm` |

### 启动前检查

- [ ] 已安装 `requirements-dev.txt` 中的开发依赖。
- [ ] `python -m compileall -q src tests` 通过。
- [ ] `python -m ruff check src tests` 通过。
- [ ] `python -m mypy` 通过。
- [ ] `python -m pytest -q` 默认测试通过。
- [ ] 若验证音频混合或打包，确认 `ffmpeg/ffmpeg.exe` 存在。
- [ ] PyCharm 运行 `src/main.py` 后系统托盘出现 QuickRec 图标。
- [ ] 托盘菜单包含全屏录制、区域录制、窗口录制、设置、打开保存文件夹、退出。

### 注意事项

- v1.4 不新增用户可见功能，重点验证稳定性、工程化、异常路径和回归行为。
- 默认自动化测试不包含 `hardware` 和 `packaging` marker。
- `hardware` 测试需要真实 Windows 桌面捕获能力；在无真实桌面会话或远程/受限环境中可能失败。
- 当前 worktree 未包含 `ffmpeg/ffmpeg.exe`，打包前需手动放置。
- 打包前可优先在 PyCharm 中运行项目完成手动功能验证。

## v1.4 已知限制和风险

1. **FFmpeg 二进制未随 git 提交**：打包前必须手动放置 `ffmpeg/ffmpeg.exe`。
2. **hardware 测试依赖真实桌面会话**：受限环境中 `ScreenCapturer` 可能返回 `None`，需在本机桌面验证。
3. **打包体积目标调整为稳定性优先**：dxcam 默认捕获链路依赖 `cv2`，当前恢复 OpenCV 后本地打包结果为 257.74MB；已排除 OpenCV 自带 `opencv_videoio_ffmpeg*.dll`，后续体积优化不得破坏录制稳定性。
4. **特殊窗口兼容性不是 v1.4 新功能目标**：游戏/UWP/DWM 自定义渲染窗口仍可能不可录制；v1.4 只要求失败稳定、提示清晰、不崩溃。
5. **磁盘低空间手动测试需要专门环境**：建议使用 U 盘、小分区或可控测试目录，不建议在系统盘根目录做极限测试。
